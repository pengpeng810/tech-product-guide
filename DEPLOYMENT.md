# Deployment Notes

## Current Site

The static site is in `site/`.

Open this file locally:

`site/index.html`

## Recommended First Deployment Path

Use a static hosting platform such as Vercel, Netlify, Cloudflare Pages, or GitHub Pages.

For the first version, the site does not need a backend, database, checkout, or affiliate system. It only needs public HTML pages that can be crawled and cited.

## Permissions Needed From the Owner

To publish the site, provide one of the following:

1. Access to a hosting account such as Vercel, Netlify, Cloudflare Pages, or GitHub Pages.
2. A domain name or permission to register one.
3. Permission to connect the domain DNS to the chosen host.

Before any public publish, the final deploy button should be confirmed by the owner because it makes the site publicly available.

- `site/sitemap.xml`
- `site/llms.txt`

## Suggested Domain Direction

Use a neutral technology or comparison domain instead of a TicNote trademark domain. The site should be able to cover many products, not just AI voice recorders.

Good directions:

- Tech product guide
- Practical tech reviews
- AI hardware guide
- Smart productivity tools
- Tech comparison guide

Avoid using `TicNote` in the domain unless there is explicit brand authorization.
